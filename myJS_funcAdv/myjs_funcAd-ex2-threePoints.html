myjs_funcAdv: exercise Остаточные параметры (...)

На вход функции showName() получает аргументы-строки из которых первые два - это имя и фамилия. Нам нужно отделить остальные аргументы от первых двух и поместить их в отдельный массив. Так же, надо показать количество элементов в этом массиве.
Объясните разницу между структурой Остаточные параметры и псевдомассивом arguments?

Решение:

Как известно, вызывать функцию можно с любым количеством аргументов независимо от того, как она была определена. Для решения этой задачи используем структуру 'остаточные параметры', которые могут быть обозначены через три точки (...), что буквально значит: «собери оставшиеся параметры и положи их в массив».
Таким образом, в аргументах функции, первые два параметра обозначим как имя и фамилия а для остальных параметров определим название массива, например 'titles', с тремя точками. Остаточные параметры собирают все остальные аргументы, поэтому бессмысленно писать что-либо после них. Это вызовет ошибку. Далее, в контексте выполнения выведем в консоль отдельно имя и фамилию, и отдельно элементы массива с остаточными параметрами, так же и количество элементов используя свойство массивов arr.length вот так:


function showName(firstName, lastName, ...titles) {
console.log( firstName + ' ' + lastName ); // Юлий Цезарь

// Оставшиеся параметры пойдут в массив titles = ['Консул', 'Император']
console.log( titles[0] ); // Консул
console.log( titles[1] ); // Император
console.log( titles.length ); // 2
}

showName('Юлий', 'Цезарь', 'Консул', 'Император');


Все аргументы функции находятся в псевдомассиве arguments под своими порядковыми номерами.
Раньше в языке не было остаточных параметров, и получить все аргументы функции можно было только с помощью arguments. Этот способ всё ещё работает, мы можем найти его в старом коде.


function showName() {
console.log( arguments.length ); // 2
console.log( arguments[0] ); // Сигизмунд
console.log( arguments[1] ); // Евстафиевич

// Объект arguments можно перебирать
// for (let arg of arguments) alert(arg); // Сигизмунд, Евстафиевич
}

showName('Сигизмунд', 'Евстафиевич'); // 2, Сигизмунд, Евстафиевич

showName('Модест'); // 1, Модест, undefined - второго аргумента нет


Хотя arguments похож на массив, и его тоже можно перебирать, это всё же не массив. Он не поддерживает методы массивов. К тому же, arguments всегда содержит все аргументы функции — мы не можем получить их часть. А остаточные параметры позволяют это сделать.
Соответственно, для более удобной работы с аргументами лучше использовать остаточные параметры.
