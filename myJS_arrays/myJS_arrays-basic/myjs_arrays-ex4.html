myjs-arrays: basic_exercises: перебор элементов массива циклами for/for..of

Мы имеем массив элементов 'fruits'. Осуществите перебор элементов этого массива? На массиве 'tropicalFruits' покажите другой способ перебора элементов? Есть ли ещё методы перебора?

let fruits = ['Яблоко', 'Груша', 'Слива', 'Черешня', 'Абрикос', 'Персик'];
let tropicalFruits = ['Ананас', 'Гуава', 'Личи', 'Папая', 'Манго', 'Авокадо'];


Решение:

Одним из самых старых способов перебора элементов массива является цикл for по цифровым индексам:


let fruits = ['Яблоко', 'Груша', 'Слива', 'Черешня', 'Абрикос', 'Персик'];

for (let i = 0; i < arr.length; i++) {
console.log( arr[i] );
}


Но для массивов возможен и другой вариант цикла, for..of по значениям:


let tropicalFruits = ['Ананас', 'Гуава', 'Личи', 'Папая', 'Манго', 'Авокадо'];


// проходит по значениям
for (let fruit of tropicalFruits) {
console.log( fruit );
}


Цикл for..of не предоставляет доступа к номеру текущего элемента, только к его значению, но в большинстве случаев этого достаточно. А также это короче.


Технически, так как массив является объектом, можно использовать и вариант for..in:


let berries = ['Арбуз', 'Виноград', 'Клубника', 'Агрус'];

for (let key in berries) {
console.log( berries[key] ); // ['Арбуз', 'Виноград', 'Клубника', 'Агрус']
}


Но на самом деле это – плохая идея. Существуют скрытые недостатки этого способа:

1. Цикл for..in выполняет перебор всех свойств объекта, а не только цифровых.
В браузере и других программных средах также существуют так называемые «псевдомассивы» – объекты, которые выглядят, как массив. То есть, у них есть свойство length и индексы, но они также могут иметь дополнительные нечисловые свойства и методы, которые нам обычно не нужны. Тем не менее, цикл for..in выведет и их. Поэтому, если нам приходится иметь дело с объектами, похожими на массив, такие «лишние» свойства могут стать проблемой.
2. Цикл for..in оптимизирован под произвольные объекты, не массивы, и поэтому в 10-100 раз медленнее. Увеличение скорости выполнения может иметь значение только при возникновении узких мест. Но мы всё же должны представлять разницу.


В общем, не следует использовать цикл for..in для массивов.
