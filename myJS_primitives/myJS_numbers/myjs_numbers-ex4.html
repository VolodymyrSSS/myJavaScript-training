myJS-primitives: numbers: exercise

Взгляните на код ниже. Этот цикл – бесконечный. Он никогда не завершится, почему?

let i = 0;
while (i != 10) {
i += 0.2;
}


Ответ: Потому что i никогда не станет равным 10.

Для проверки удовлетворения условия, чтобы код остановился, необходимо проверить, будет ли хоть одно значение равно 10.
Так как мы имеем шаг прибавления - десятичную дробь то для этого переделаем немного код и изменим значения диапазона на десятичные дроби и также изменим диапазон от 9.8 до 10.2. чтобы внутри было значение 10.


let i = 0;
while (i < 11) {
i += 0.2;
if (i > 9.8 && i < 10.2) console.log( i );// 9.999999999999996 и 10.199999999999996
}


Ни одно из этих чисел не равно 10.
Это происходит из-за потери точности, при прибавлении таких дробей как 0.2


Вывод: избегайте проверок на равенство при работе с десятичными дробями!
