Basic JS: логический оператор '&&' (AND)

Сейчас у нас время: 12.30. Напешите код который выведет 'Время пить чай 12.30'.

Решение:

В традиционном программировании && (AND) возвращает true, если оба аргумента истинны, а иначе – false.
Существует всего четыре возможные логические комбинации c логическим оператором && (AND), по русски 'И':



alert( true && true ); // true
alert( false && true ); // false
alert( true && false ); // false
alert( false && false ); // false


Оператор 'И' выполняет следующие действия:
Вычисляет операнды слева направо.
Каждый операнд конвертирует в логическое значение.
Только если оба результата true, останавливается и возвращает исходное значение этого операнда.

То есть, логический оператор && возвращает первое ложное значение.


Как и в случае с 'ИЛИ', любое значение допускается в качестве операнда 'И':
if (0 && "no matter what") { // вычисляется как false && true alert( 'не сработает, так как результат ложный' ); }


вот, ещё несколько примеров:


// Если первый операнд истинный, 'И' возвращает второй:
console.log( 1 && 0 ); // 0
console.log( 'Sally' && undefined ); // undefined

// Если первый операнд ложный, 'И' возвращает его. Второй операнд игнорируется console.log( 0 && 1 ); // 0
console.log( null && 5 ); // null


Итак, для решения задачи используем оператор 'И':



let hour = 12;
let minute = 30;

if (hour == 12 && minute == 30) {
console.log( 'The time to drink a tea 12:30' );
}


Можно передать несколько значений подряд.
При нескольких подряд операторах:
result = value1 && value2 && value3; - && находит первое ложное значение на котором остановились вычисления
При этом, оператор && выполняет следующие действия:
Вычисляет операнды слева направо.
Каждый операнд преобразует в логическое значение.
Если результат false, останавливается и возвращает исходное значение этого операнда.
Если все операнды были истинными, возвращается последний.

Другими словами, && возвращает первое ложное значение. Или последнее, если ничего не найдено.



console.log( 1 && 2 && null && 3 ); // null, так как это первое ложное значение, на котором остановились вычисления, оно и возвратится.

console.log( 1 && 2 && 3 ); // 3, так как когда все значения верны, возвращается последнее


Вышеуказанные правила схожи с поведением 'ИЛИ'. Разница в том, что 'И' возвращает первое ложное значение, а 'ИЛИ' –  первое истинное.


Приоритет оператора && больше, чем у ||
Приоритет оператора 'И' больше, чем 'ИЛИ', так что он выполняется раньше.
Таким образом, код a && b || c && d по существу такой же, как если бы выражения && были в круглых скобках: (a && b) || (c && d).


Как и оператор 'ИЛИ', 'И' иногда может заменять if, к примеру:


let x = 1;

(x > 0) && alert( 'Greater than zero!' );


Действие в правой части && выполнится только в том случае, если до него дойдут вычисления. То есть, alert сработает, если в левой части (x > 0) будет true.
Однако, как правило, вариант с if лучше читается и воспринимается. Он более очевиден, поэтому лучше использовать его. Вот посмотрите:



let x = 1;

if (x > 0) {
alert( 'Greater than zero!' );
}
