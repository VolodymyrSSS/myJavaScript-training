Basic JS: function-expression

Мы имеем Объявление Функции (или Function Declaration) с названием 'sayHi'. Переделайте эту функцию в другой вид - Функциональное Выражение (или Function Expression);
Потом, раздельно, выведите в консоль результат работы этой функции, а на экран строковое представление этой функции.
Далее скопируйте функцию в другую переменную и выведите в консоль результат вызова копии.

function sayHi() {
console.log( 'Hallo' );
};


Решение:


Существует ещё один синтаксис создания функций, который называется Function Expression (Функциональное Выражение).
Оно выглядит вот так:

let sayHi = function() {
console.log( 'Hallo' );
};


В коде выше функция создаётся и явно присваивается переменной, как любое другое значение. По сути без разницы, как мы определили функцию, это просто значение, хранимое в переменной sayHi.
Смысл обоих частей кода одинаков: 'создать функцию и поместить её значение в переменную sayHi'.


Теперь давайте выведем в консоль результат работы этой функции а на экран - код функции
При этом для второго, вызов функции sayHi осуществляем без круглых скобок.
То есть код выведет строковое представление функции, которое является её исходным кодом.


sayHi();

alert(sayHi); -нету круглых скобок


В JavaScript функции – это значения, поэтому мы и обращаемся с ними, как со значениями.
Мы можем скопировать функцию в другую переменную:


let func = sayHi; -копируем, внимание, так же нету круглых скобок!

// Теперь функция может быть вызвана с помощью обеих переменных sayHi() и func().

sayHi(); -выведет Hallo
func(); -выведет Hallo


В строке где мы скопировали её значение в переменную func - обратите внимание ещё раз!: нет круглых скобок после sayHi. Если бы они были, то выражение func = sayHi() записало бы результат вызова sayHi() в переменную func, а не саму функцию sayHi.


У вас мог возникнуть вопрос: Почему в Function Expression ставится точка с запятой ; на конце, а в Function Declaration нет:
Нет необходимости в ; в конце блоков кода и синтаксических конструкций, которые их используют, таких как if { ... }, for { }, function f { } и т.д.
Function Expression использует внутри себя инструкции присваивания let sayHi = ...; как значение. Это не блок кода, а выражение с присваиванием. Таким образом, точка с запятой не относится непосредственно к Function Expression, она лишь завершает инструкцию.
