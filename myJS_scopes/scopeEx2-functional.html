Function scope

Scopes: сутність Function scope

Дайте відповідь, що таке область видимості функції?

Відповідь:


Саме слово область видимості - визначає доступність змінних, об’єктів і функцій з різних частин коду.
До ES6 (2015) JavaScript мав лише глобальну область видимості та область видимості функцій. І отже змінні, які оголошені всередині функції, створюють область видимості функції або функційну область видимості.
Змінні, визначені всередині функції, недоступні (видимі) ззовні функції. Час життя змінної JavaScript починається з моменту її оголошення. Функціональні (локальні) змінні видаляються, коли функція завершується. А от у веб-браузері глобальні змінні видаляються, коли ви закриваєте вікно (або вкладку) браузера.

Змінні, оголошені за допомогою var, let і const, дуже схожі, коли оголошені всередині функції. Усі вони мають функційну область видимості: 

          function myFunction() {
            var carName = "Volvo";   // Function Scope
          }

          function myFunction() {
            let carName = "Volvo";   // Function Scope
          }

          function myFunction() {
            const carName = "Volvo";   // Function Scope
          }


Як відомо, ES6 ввів для використання важливі ключові слова для оголошення змінних в JavaScript: let і const. Ці два ключові слова забезпечують впровадження в JavaScript розуміння та використання блочної області - змінні, оголошені всередині блоку {}, не можуть бути доступні поза блоком:

          {
            let x = 2;
          }
          // x не може бути використана/доступна тут


Змінні, які оголошені за допомогою ключового слова var, не можуть мати блочної області. Такі змінні, хоч і оголошені всередині блоку {}, можна отримати ззовні блоку: 

          {
            var x = 2;
          }
          // x може бути використана/доступна


Необхідно зазначити, що в JavaScript змінні, оголошені в функції, стають лоакльними для цієї функції. А локальні змінні мають функційну область видимості. До них можна отримати доступ лише зсередини функції: 

        // тут код не може використовувати змінну carName

        function myFunction() {
          let carName = "Volvo";
          // тут код може використовувати змінну carName
        }

        // тут код не може використовувати змінну carName


Оскільки локальні змінні розпізнаються лише всередині своїх функцій, змінні з однаковими іменами можна використовувати в різних функціях.
Локальні змінні створюються, коли функція запускається, і видаляються, коли функція завершується.
