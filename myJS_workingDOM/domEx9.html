DOM (data object model)

Working with DOM: using properties: innerHTML, tagName, firstChild
properties for text data

Використання властивості innerHTML.
В нас показана частина HTML-сторінки, на якій представлено код в тезі script.
Поясніть, що буде виведено в результаті роботи скірпта? 


          // Вихідні дані:

          <html>
          <body>
            <script>
              let body = document.body;

              body.innerHTML = "<!--" + body.tagName + "-->";

              console.log( body.firstChild.data ); // що буде виведено?
            </script>
            .... інший код....
          </body>
          </html>


Рішення: буде виведено BODY



З теорії відомо, що отримавши DOM-вузел, ми можемо дізнатись назву його тега використавши властивість або nodeName, або tagName.
Властивість tagName є тільки в елеменів Element;
Властивість nodeName належить будь-яким вузлам Node;
Якщо ми маєм справу тільки з елементами, то можна використовувати як tagName так і nodeName, тут немає різниці. Крім того, в HTML-режимі обробки браузером, значення tagName/nodeName завжди записані у верхньому регістрі незалежно від того як записаний тег в HTML.
Отже, ми отримаємо назву тегу для елемента body.tagName == "BODY" і далі заміняємо вміст body на коментар, який буде мати вид: 


          let body = document.body; // отримаєм для вузла-елемента тег BODY

          body.innerHTML = "<!--" + body.tagName + "-->"; // <!--BODY-->


Цей коментар тепер є першим і єдиним потомком для body.firstChild. Значення властивості data для елемента-коментаря - це його вміст/контент (всередині <!--...-->) - це "BODY".
